# Debug Mode Rules

## Debugging Focus
- Problem solving and debugging focused
- Systematic debugging approach
- Root cause analysis

## Laravel Debugging Tools
- **Laravel Telescope**: Query monitoring, exception tracking
- **Debug Bar**: SQL query analysis, performance metrics
- **Log Analysis**: Laravel logs, error tracking
- **Artisan Commands**: cache:clear, config:clear, optimize:clear

## Financial System Debugging
- **Transaction Issues**: Database transaction rollbacks
- **Balance Discrepancies**: Account balance calculation errors
- **Payment Gateway**: API response analysis, webhook failures
- **Lead Assignment**: Admin assignment logic, status updates
- **Authentication**: Login issues, 2FA problems

## Debugging Process
1. **Reproduce**: Recreate the issue consistently
2. **Logs**: Check Laravel logs, database queries
3. **Trace**: Follow code execution path
4. **Test**: Create minimal test case
5. **Fix**: Apply targeted solution
6. **Verify**: Ensure fix doesn't break other features

## Common Issues
- **Memory**: Large dataset processing
- **Performance**: N+1 queries, missing indexes
- **Security**: Authentication bypass, privilege escalation
- **API**: External service timeouts, rate limiting
- **Livewire**: Component state management, wire:model issues

## Testing Strategy
- **Unit Tests**: Isolated component testing
- **Feature Tests**: End-to-end user flows
- **Database Tests**: Transaction integrity
- **Mock Testing**: External API failures
